# CLAUDE.md — せーの！神輿バトル オンライン対戦版

このリポジトリは、日本の祭りをテーマにしたリアルタイム対戦ゲームです。
リポジトリの所有者はエンジニアではありません。**説明はすべて日本語で、専門用語を避けて**行ってください。

## プロジェクト概要

- 複数プレイヤーが「連（チーム）」を組み、「せーの！」の合図に合わせて同時にタップし、シンクロ率で対戦するゲーム
- サーバー: `server.js`（Node.js / Express + ws）
- クライアント: `public/index.html`（単一ファイル。Three.js r128 による3D演出込み、約166KB）
- デプロイ: GitHub の main ブランチに反映されると Render が自動でデプロイする

## 絶対に守るルール

1. **サーバー権威の設計を崩さない。** 勝敗判定・パワー計算・コイン残高・ガチャ抽選・マッチングは必ずサーバー側（server.js）で行う。クライアント側に判定や残高計算を移してはならない（チート対策のため）。
2. **クライアントとサーバーの定数を一致させる。** タップ受付時間（TAP_DURATION_MS = 5000ms）や判定しきい値（MIRACLE≥1.0 / PERFECT≥0.8 / GREAT≥0.6 / NICE≥0.4）を片方だけ変更しない。変更する場合は両方を同時に変更し、テストで確認する。
3. **WebSocketメッセージの形式（プロトコル）を互換性なく変えない。** メッセージの種類（register / lobby / round / tap / roundResult / battle / settle / gacha など）のフィールド名を変更・削除する場合は、server.js と public/index.html の両方を必ず同時に修正する。
4. **ギャンブルを連想させる言葉を使わない。** 「賭け」「カジノ」「ベット」「換金」等はUI・コード・コメントのいずれにも入れない。ガチャは「衣装ガチャ」、コインはゲーム内通貨（換金不可）として扱う。
5. **コインは勝利時のみ付与。** 勝利 +20、シンクロ貢献 +5×成功ラウンド数。参加・広告視聴・敗北でコインを配る変更は、所有者から明示的な指示がない限り行わない。
6. **UIは日本語、スマホ縦画面（390×844想定）を基準にする。**
7. **既存の3D演出・音・アニメーションのコードは、依頼と無関係なら触らない。**

## 作業の進め方（毎回必ず）

1. 変更前に、関係するファイルを読んで現状を把握する
2. 変更を加える
3. **テストを実行して全て合格することを確認する（下記コマンド）**
4. テストが通らない変更は push しない。直してから push する
5. 完了報告は日本語で、「何がどう変わったか」「遊ぶ人にどう見えるか」を先に書く

## テストコマンド

```bash
npm install
node test_e2e.js       # サーバーの対戦ロジック検証（47チェック）
node test_browser.js   # 実際の画面を動かす通しテスト
```

- `test_e2e.js` は PvP対戦・協力プレイ・連戦・ガチャ・不正タップ拒否を検証する
- `test_browser.js` は jsdom 上で実際のクライアントを動かし、登録→対戦→結果→ガチャまで通す
- タイミング系の定数（CUE_LEAD 等）は環境変数で上書き可能。テストは高速設定で動く

## 運用上の注意（回答時に考慮すること）

- Render 無料プランでは、デプロイのたびにプレイヤーデータ（コイン・衣装）がリセットされる。データ保存に関する要望が出たら、有料の永続ディスク追加または外部データベース化を提案する
- 無料プランは15分アクセスがないとサーバーが休止し、次のアクセス時に起動へ約1分かかる
- ルーム状態はメモリ管理のため、サーバーは1台構成が前提。複数台への拡張はRedis等が必要になる旨を伝える
